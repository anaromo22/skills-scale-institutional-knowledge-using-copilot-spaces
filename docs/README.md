# OctoAcme Project Management Docs

Welcome to the OctoAcme project management process documentation. This README provides an overview of how OctoAcme runs projects and serves as a table of contents for all process documents in the docs/ folder.

## Purpose

This documentation centralizes project management knowledge to:
- Provide equal access to processes, decisions, and rationale across the team
- Convert team insights into searchable, versioned artifacts
- Accelerate onboarding and reduce single-person dependency risk
- Enable consistent, repeatable project execution

## OctoAcme Project Management Overview

Our approach is built on five core principles:

- **Customer-first:** Prioritize customer value and usability
- **Iterative delivery:** Deliver in small, testable increments with regular stakeholder feedback
- **Clear ownership:** PM and Product Lead are accountable for outcomes
- **Data-informed:** Use metrics and insights to drive decisions
- **Psychological safety:** Encourage open feedback and continuous learning

### Project Lifecycle Stages

OctoAcme projects progress through five key stages:

1. **Initiation:** Validate business need, identify stakeholders, create a one-pager, and make a go/no-go decision
2. **Planning:** Define scope, build the backlog, establish milestones, identify risks, and secure resources
3. **Execution:** Build and test features, track progress, manage blockers, and iterate based on feedback
4. **Release:** Deploy to production, verify functionality, and communicate changes to stakeholders
5. **Retrospective:** Capture learnings, document improvements, and feed insights back into process refinement

### Key Roles

- **Project Manager (PM):** Coordinates delivery, manages schedules, risks, and communications
- **Product Manager (PdM):** Defines outcomes, prioritizes the backlog, and measures success
- **Developers:** Implement features, collaborate on design, and ensure code quality
- **QA/Testing:** Validate quality and acceptance criteria
- **Stakeholders:** Provide inputs, approvals, and business context
